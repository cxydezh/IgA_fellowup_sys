# 端口5000占用情况说明

## 检查结果

### 占用端口的程序
- **程序名称**：`PDDPrintClient.exe`
- **程序说明**：拼多多打印客户端
- **进程ID**：29656
- **父进程ID**：13308
- **启动时间**：2025-11-09 8:03:41

### 详细信息
- **进程类型**：打印客户端程序
- **占用端口**：5000
- **监听地址**：0.0.0.0:5000（所有网络接口）
- **状态**：LISTENING（正在监听）

## 为什么会出现这个问题？

1. **拼多多打印客户端**是一个独立的程序，用于处理打印任务
2. 该程序默认使用端口5000进行通信
3. 当该程序运行时，会占用端口5000
4. 我们的Flask应用如果也尝试使用端口5000，就会发生冲突

## 解决方案

### 方案1：使用其他端口（已实施）
- Flask应用已配置为使用端口5001
- 这是最简单且不影响其他程序的方案
- 当前访问地址：http://localhost:5001

### 方案2：关闭拼多多打印客户端
如果不需要使用拼多多打印客户端，可以：
1. 打开任务管理器
2. 找到 `PDDPrintClient.exe` 进程
3. 结束该进程
4. 然后Flask应用就可以使用端口5000了

### 方案3：修改拼多多打印客户端端口
如果必须使用端口5000，可以尝试：
1. 查看拼多多打印客户端的配置
2. 修改其使用的端口号
3. 但这需要修改第三方程序的配置，可能比较复杂

## 当前状态

✅ Flask应用已配置为使用端口5001运行
✅ 不会与拼多多打印客户端冲突
✅ 应用可以正常运行

## 如何检查端口占用

使用以下命令检查端口占用：
```bash
# Windows
netstat -ano | findstr :5000

# 查看进程详情
wmic process where "ProcessId=进程ID" get ProcessId,Name,CommandLine,ExecutablePath
```

## 注意事项

1. 如果以后需要将Flask应用改回端口5000，需要先关闭拼多多打印客户端
2. 端口5001是安全的，不会与其他常见服务冲突
3. 如果端口5001也被占用，可以尝试5002、5003等其他端口
